**Punti delicati / rischi noti**
- Il codice C# non è stato compilato con Unity ancora, quindi potrebbero emergere errori di compatibilità API o serializzazione al primo import.
- `SimulationConfigLoader.LoadDefault()` usa path filesystem candidati; potrebbe dover essere adattato quando il progetto gira davvero dentro Unity o su Android build.
- I test C# EditMode esistono in [SimulationEngineTests.cs](/C:/Pampa's%20skylines/unity/Assets/Game/Tests/EditMode/SimulationEngineTests.cs), ma non sono ancora stati eseguiti.
- Non ci sono file `.meta` generati da Unity finché il progetto non viene aperto nell’editor.
- Il backend è file-based; Firebase è solo adapter opzionale, non operativo end-to-end.
